Guía de Prompts UBITS

Prompts para usar en Cursor AI. Sin conocimiento técnico necesario, solo copia, pega y personaliza.

Cómo usar esta guía

1
Copia el prompt
Haz clic en el botón "Copiar" del prompt que necesites. Se copiará automáticamente a tu portapapeles.
2
Personaliza el prompt
Reemplaza los textos entre corchetes [TEXTO] con tu información específica antes de usar el prompt.
3
Pega en Cursor AI
Abre Cursor AI, pega el prompt en el chat y presiona Enter. Cursor hará todo el trabajo por ti.
4
Revisa y ajusta
Revisa los cambios que hizo Cursor y pide ajustes si es necesario. ¡Es así de fácil!

1. Clonar el repositorio UBITS

Usa este prompt para obtener el UBITS Playground completo en tu computadora por primera vez.

Quiero trabajar contigo usando el UBITS Playground que está en este repositorio: https://github.com/DavidVegaM91/template-ubits.git 1. Importa el repositorio. 2. Lee los archivos README.md y .cursor/rules/cursor-rules.mdc para entender el contexto del proyecto. 3. Confírmame que comprendiste la configuración y pregúntame qué página quiero editar o qué nuevo flujo quiero crear.

2. Crear página de inicio del proyecto

Usa este prompt para configurar la página principal de tu proyecto UBITS.

Necesito editar index.html. Esta es una tarea grande que necesito que dividas los pasos que consideres necesarios y cumplas cada uno sin omitir nada: 1. **CONTENIDO PRINCIPAL:** - Edita la sección de Contenido principal y reemplázala con: [inserta tu idea aquí lo más completa posible]. - Asegúrate de cumplir con todos los requerimientos - Usa la estructura modular de secciones según como lo requieras: section-single (1 columna), section-dual (2 columnas), section-triple (3 columnas), section-quad (4 columnas) - Cada widget debe tener un nombre descriptivo en el código 2. **VERIFICACIÓN:** - Confirma que todos los tokens ubits-colors estén aplicados correctamente y no haber usado colores hardcodeados - NO hardcodees componentes que ya existen en la carpeta components (button, alert, card content, input, toast). - Si se usaron botones confirma que usaste los botones de ubits usa ubits-button - **SIEMPRE importa CSS** de cada componente usado - **VALIDA que TODAS las clases de tipografía que usaste hacen parte de la lista oficial de UBITS-TYPOGRAPHY-GUIDE.md** NUNCA inventes clases como ubits-h1, ubits-h2, ubits-title, ubits-text, ubits-paragraph** - Asegúrate de que la interfaz sea responsive y funcione en modo claro/oscuro Si tienes dudas sobre mi idea, por favor hazme preguntas para darte mas detalles!!!!

4. Actualizar el repositorio UBITS

Usa este prompt cuando quieras obtener las últimas actualizaciones del template sin perder tus cambios personalizados.

1. Haz un backup de mis archivos personalizados (todos los que he modificado) 2. Obtén la versión más reciente del template desde https://github.com/DavidVegaM91/template-ubits 3. Compara archivo por archivo y actualiza solo: - Archivos del sistema UBITS (components/, ubits-colors.css, ubits-typography.css, fontawesome-icons.css) - Archivos de documentación (documentacion.html, componentes.html, validador-ubits.html, guia-prompts.html) - Páginas de componentes (sidebar.html, subnav.html, tab-bar.html, button.html, alert.html, card-content.html) - Guías de diseño (colores.html, tipografia.html, iconos.html) - Archivos de configuración (.cursor/rules/cursor-rules.mdc, README.md) 4. NO toques mis archivos personalizados (index.html, mis páginas creadas, mis modificaciones) 5. Verifica que todo funcione correctamente Mi objetivo es tener las últimas mejoras del UBITS Playground manteniendo mis personalizaciones.
